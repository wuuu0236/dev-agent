# RAG 技术详解

## RAG 是什么

RAG（Retrieval-Augmented Generation，检索增强生成）是让 LLM 在回答问题时先到知识库中检索相关信息，再基于检索到的内容生成答案。这解决了 LLM 的两个核心问题：知识截止日期（LLM 不知道训练后的新知识）和幻觉（LLM 可能编造不存在的事实）。

## RAG 和普通搜索的区别

| 维度 | RAG | 普通关键词搜索 |
|------|-----|---------------|
| 匹配方式 | 语义理解（向量相似度） | 关键词精确匹配 |
| 容错能力 | 不同表述也能匹配到相关内容 | 换个词就搜不到 |
| 结果形式 | 自然语言答案（LLM 生成） | 文档链接列表 |
| 适用场景 | 复杂问题、需要综合信息 | 找具体文件、精确匹配 |

RAG 底层用向量相似度来检索：文字转成向量后，意思相近的文字向量也相近，所以即使表述不同也能找到相关内容。

## RAG 全链路五步

RAG 处理流程包含以下步骤：

1. **文档分段（Chunking）**：把长文档切成小段（chunk），每段 200-500 字。太大会检索不精准，太小会丢失上下文。相邻段之间可以有 overlap（重叠），防止关键信息被切断。

2. **向量化（Embedding）**：用 Embedding 模型（本项目用 text2vec-base-chinese）把每一段文字转换成向量（一串数字）。语义相近的文字，向量也相近。

3. **存入向量库（Indexing）**：把所有向量存起来。专业方案用 Chroma、Milvus、Pinecone，本项目用 NumPy 自实现以理解原理。

4. **检索（Retrieval）**：用户提问后，同样用 Embedding 模型把问题转成向量，然后在向量库中计算余弦相似度，找出最相关的 top_k 段文档。

5. **生成（Generation）**：把检索到的文档和用户问题一起发给 LLM，LLM 基于文档内容生成答案，避免编造。

## 余弦相似度原理

余弦相似度衡量两个向量的夹角：值越接近 1 越相关，越接近 0 越不相关。计公算式为 cos_sim = (A·B) / (|A| × |B|)，即两个向量点积除以各自模长的乘积。这是向量检索的核心数学原理。

## 向量数据库的作用

传统数据库存的是结构化数据（数字、文本），向量数据库存的是向量（高维数组），专门做相似度搜索。Chroma 和 Milvus 本质上就是存向量 + 建索引 + 快速做余弦相似度搜索，本项目用 NumPy 实现了一个简化版来理解底层原理。
